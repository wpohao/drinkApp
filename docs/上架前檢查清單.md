# 糖手帳（DrinkSugar）— Google Play 上架前檢查清單

> 對應實作計畫 Task 19。V1 送審前逐項確認。

## 1. App 圖示與商店素材
- [ ] 用 Android Studio「New → Image Asset」產生 adaptive icon（覆蓋 `res/mipmap-*`；目前為暫用向量圖示）。
- [ ] Play 商店圖示 512×512（PNG）。
- [ ] 功能圖片（Feature graphic）1024×500。
- [ ] 手機螢幕截圖至少 2 張（今日頁、歷史頁）。

## 2. AdMob 換正式 ID
- [ ] `AndroidManifest.xml` 的 `com.google.android.gms.ads.APPLICATION_ID`：測試 `ca-app-pub-3940256099942544~3347511713` → **正式 App ID**。
- [ ] `ads/AdBanner.kt` 的 `adUnitId`：測試 `ca-app-pub-3940256099942544/6300978111` → **正式橫幅單元 ID**。
- [ ] 實機確認正式廣告能載入（新帳號可能需等 AdMob 審核）。

## 3. 簽章與 release 建置
- [ ] 產生 upload keystore（`.jks`）。
- [ ] `app/build.gradle.kts` 設 `signingConfigs`；金鑰資訊放 `keystore.properties`（**已在 .gitignore 排除，勿進 git**）。
- [ ] 啟用 R8：release `isMinifyEnabled = true` 並驗證無 keep 規則問題。
- [ ] `./gradlew bundleRelease` 產出 `app-release.aab`，上傳 Play Console 內部測試軌道。

## 4. 版本
- [ ] `versionCode` / `versionName` 確認（首版 1 / 1.0）。

## 5. Google Play Console
- [ ] 建立 App，填 metadata：名稱、簡短/完整說明、關鍵字「手搖、糖分、控糖、飲料、熱量、手搖日記」。
- [ ] 說明與 App 內皆含「**數值為參考估算，非醫療建議**」。
- [ ] 內容分級問卷。

## 6. 隱私與權限
- [ ] Play「資料安全」表單：本 App 資料全存本機；AdMob 使用廣告識別碼需申報。
- [ ] 隱私政策網址（可用 GitHub Pages 放一頁）。
- [ ] 權限用途說明：`POST_NOTIFICATIONS`（提醒）、`INTERNET`（廣告）。

## 7. 實機測試
- [ ] 記一杯（含冰塊/星級/加料/手動）→ 今日糖分與清單更新。
- [ ] 歷史四 tab：趨勢、日曆（控糖顏色）、圓餅、戰績卡分享。
- [ ] 通知：接近/超標即時通知、每日補記提醒；Android 13+ 權限流程。
- [ ] 廣告：底部測試橫幅顯示、不誤觸。
